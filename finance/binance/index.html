Top 6 Representative Compliance Cases

每個案例呈現如何為受監管金融產品設計產品規則、流程狀態、內部角色與風險控管。

Case 01

KYC / AML 篩查流程

設計串接身份驗證、PEP 檢查、制裁名單篩查、人工審核、補件、核准、拒絕與交易啟用的開戶狀態。

  • 角色：申請客戶、Compliance reviewer、Operations agent。
  • 邏輯：provider response mapping、risk hit、queue routing、權限控管。
  • 脈絡：Veriff AML Screening、AUSTRAC CDD
Case 02

適性評估與資格決策引擎

將 CFD 適性要求轉換為確定性資格規則、拒絕邏輯、風險揭露 gate、覆核狀態與產品權限控管。

  • 角色：申請客戶、Compliance、Product、Engineering。
  • 邏輯：pass、fail、restrict、resubmit、manual review、audit trail。
  • 脈絡：ASIC RG227、FCA Appropriateness
Case 03

多區域合規規則矩陣

建立以 jurisdiction 為基礎的開戶邏輯，涵蓋國家限制、身份文件要求、產品權限、支付方式與客戶風險等級。

  • 角色：Product、Compliance、Operations、市場團隊。
  • 邏輯：國家、居住地、文件類型、產品可用性、支付可用性。
  • 脈絡：FATF Recommendations
Case 04

支付與錢包驗證控管

設計支付方式、錢包歸屬、第三方支付風險、客戶風險等級、hold、reject 與 review routing 的資金風控。

  • 角色：客戶、Risk、Operations、支付支援團隊。
  • 邏輯：KYC dependency、支付資格、錢包匹配、hold 與 release states。
  • 脈絡：Chainalysis KYT、TRM Wallet Screening
Case 05

資金來源與 EDD 審核流程

定義加強審核觸發條件、文件收集規則、案件升級、交易 hold、放行條件，以及 Operations 與 Compliance 之間的狀態回寫。

  • 角色：高風險客戶、審核人員、Compliance lead。
  • 邏輯：trigger table、要求文件、hold、escalate、approve、reject、audit log。
  • 脈絡：AUSTRAC Source of Funds、Enhanced CDD
Case 06

跟單交易風險控管

為高風險交易產品建立 guardrails，包含資格檢查、風險提示、signal review、monitoring、suspension 與使用者保護流程。

  • 角色：followers、signal providers、Risk、Product Operations。
  • 邏輯：suitability checks、warning gates、pause、stop-copy、monitoring triggers。
  • 脈絡：FCA Copy Trading
