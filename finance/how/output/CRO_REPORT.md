# FundSwap CRO Growth Optimization Report

## 完整 CRO 優化報告

### Funnel 現況

使用者路徑：

1. 進站
2. 選投資方向
3. 看基金候選
4. 展開基金理解區
5. 比較或猶豫
6. 開戶 / 小額投資 / 離開

### 最大流失點

#### 1. 進站 → 看基金

原問題：使用者一進來看到基金列表、報酬率、排序、代碼與標籤，會先覺得「我看不懂」。

目前已改善：Hero 改成「先看懂，再開始投資」，並用五大入口降低選擇壓力。

仍需強化：首屏應把「不知道怎麼選」變成主要路徑之一，而不是輔助按鈕。

高 ROI 行動：
- 保留「幫我縮小選擇」為主 CTA。
- 將「我不知道怎麼選」設計成第二高權重 CTA。
- 首屏不要再增加其他功能。

#### 2. 看基金 → 點基金

原問題：基金卡像 Excel，使用者只看到報酬率，不知道點開後能得到什麼。

目前已改善：基金卡 CTA 改為「看懂這檔」，展開後有收益、風險、情境、FAQ、Sticky CTA。

仍需強化：基金卡首層要補一句「為什麼值得點」，而不是只靠指標吸引。

高 ROI 行動：
- 每張基金卡加一句短理由：例如「適合想慢慢累積的人」。
- 預設展開第一張基金，降低空白探索成本。
- 保留 3 個核心指標即可：報酬/波動、近 3 月、1 年。

#### 3. 點基金 → 比較

原問題：使用者看完一檔後不知道是否該買，常見反應是「我再想想」。

目前已改善：展開區有 FAQ、投資情境與配置建議。

仍需強化：比較動機不足。使用者需要知道「為什麼要先存起來比較」。

高 ROI 行動：
- 將「先存起來比較」變成明確次 CTA。
- 比較清單最多顯示 3 檔，避免重新製造選擇焦慮。
- 比較內容只比三件事：適合誰、可能賺什麼、要承受什麼。

#### 4. 比較 → 開戶

最大商業流失點。

原因：
- 開戶是高摩擦行為。
- 使用者還沒建立「我知道我在買什麼」的信心。
- 直接推「開戶」容易像銷售壓力。

高 ROI 行動：
- 開戶 CTA 不要只說「開戶」。
- 改成「開戶後先小額試投」或「開戶，先把候選基金存起來」。
- 在 CTA 附近補一句信任文字：「不用一次投入大金額，先完成帳戶與風險屬性即可。」

#### 5. 猶豫 → 離開

原問題：金融產品猶豫是正常行為，但頁面沒有接住猶豫。

高 ROI 行動：
- 加入「寄給我這份候選清單」或「留下 Email / LINE，稍後再看」。
- 不要要求使用者立即投資。
- Lead capture 的文案應是「把這份基金短名單留給我」，不是「訂閱電子報」。

## 對抗式辯論

### B｜數據分析官質疑

#### 哪裡最影響 ROI？

不是基金圖表，也不是入口卡，而是「基金理解後到開戶」這段。

原因：
- 已經展開基金的人，投資意圖最高。
- 這時候如果 CTA 還只是一般開戶，會浪費高意圖流量。
- Sticky CTA 的文案與位置，是最直接影響開戶率與投資轉化率的點。

#### 哪些功能沒商業價值？

- 過多排序選項：使用者不是來玩資料表，排序多不代表更願意投資。
- 太細的圖表互動：如果不能幫使用者做決策，只會延長操作成本。
- 過多入口分類：五個已是上限，再多會降低決策速度。
- 社群式收藏功能：如果沒有連到開戶、比較、再訪提醒，商業價值低。

#### 哪些 UI 只是漂亮但沒轉化？

- 大型動畫。
- 複雜 hover 效果。
- 太多數據卡片。
- 沒有行動引導的基金故事。
- 沒有 lead capture 的 FAQ。

結論：CRO 優先順序不是「更漂亮」，而是「更快讓使用者相信自己可以開始」。

## 高 ROI 改版建議

### P0：基金展開後的 CTA 重構

現況：
- Sticky CTA 已存在，但可更貼近開戶與投資轉化。

建議：
- 主 CTA：`開戶，先小額開始`
- 次 CTA：`先存起來比較`
- 信任補句：`不用一次投入大金額，先完成帳戶與風險屬性，之後再決定投入多少。`

預期影響：
- 提高開戶 CTA 點擊率。
- 降低「一點就要買很多」的心理壓力。

### P0：比較清單只保留 3 檔

建議：
- 使用者點「先存起來比較」後，只顯示最多 3 檔。
- 比較欄位只保留：
  - 這檔適合誰
  - 可能賺什麼
  - 要承受什麼
  - 下一步 CTA

預期影響：
- 提高基金點擊率。
- 提高停留時間。
- 降低比較疲勞。

### P1：離開前 Lead Capture

建議：
- 使用者看過 2 檔基金或停留超過 60 秒後，顯示低干擾提示：
  - `把這份基金短名單留給我`
  - `稍後用 LINE / Email 繼續看`

禁止：
- 不要做全螢幕彈窗。
- 不要寫「立即訂閱」。

預期影響：
- 提高留資率。
- 提高再訪機會。

### P1：首頁 Hero CTA A/B Test

建議測試：
- A：`幫我縮小選擇`
- B：`找出適合我的基金`
- C：`我不知道怎麼選`

預期：
- 對新手流量，C 可能最高。
- 對投資熟手，A 或 B 可能最高。

### P2：SEO 內容區

建議：
- 在頁面下方加入不干擾轉化的 SEO FAQ：
  - 基金怎麼選？
  - 新手適合買基金嗎？
  - 高股息基金要注意什麼？
  - 定期定額適合誰？

預期影響：
- 提高自然流量。
- 提高長尾搜尋進站品質。

## 最終 UI 架構

### 1. Header

- 保留 FundSwap 原導覽。
- 開戶 CTA 保留，但文案可測試「開戶，先小額開始」。

### 2. Hero

- 標題：先看懂，再開始投資。
- 主 CTA：幫我縮小選擇。
- 次 CTA：我不知道怎麼選。

### 3. 五大入口

- 高股息
- 穩健成長
- 最大化收益
- AI 智能配置
- 新手低風險

角色：降低選擇焦慮，提升基金點擊率。

### 4. 問答式分流

- 3 題即可。
- 不增加更多題目。
- 目標是讓使用者得到一個方向，而不是做完整理財規劃。

### 5. 推薦 3 檔

- 不顯示太多。
- 每檔只給一句理由。
- 引導進基金展開區。

### 6. 基金 Dynamic Expand Card

順序：
1. 這檔基金在做什麼
2. 過去報酬
3. 你要承受的起伏
4. 如果你真的買了
5. 可以怎麼放進你的投資
6. 你可能會想問
7. Sticky CTA

### 7. 三檔比較清單

只在使用者主動加入後出現。

比較內容：
- 適合誰
- 可能賺什麼
- 要承受什麼
- 下一步

### 8. Lead Capture

觸發在高意圖時刻。

文案：
- 把這份基金短名單留給我。
- 稍後繼續看這幾檔基金。

### 9. Footer / Trust

保留法規資訊。

語氣：
- 清楚提醒。
- 不恐嚇。
- 不像銀行公文。
