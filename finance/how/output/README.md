# FundSwap Intelligent Fund Discovery & Conversion Optimization Project

## 目前網站結構分析

來源頁：`https://www.fundswap.com.tw/trade/funds/?fundsIndex=2&filterType=rateOfReturn&tradable=true&s=-rateOfReturn3Months&page=1`

- 技術型態：Nuxt SPA，初始 HTML 約 4.3KB，內容由前端載入。
- 字體：`Noto Sans TC`，輔以 Microsoft JhengHei 與系統字體。
- 品牌色：主橘 `#ff9664`、深橘 `#ff7e40`、淺橘 `#fff3ee/#ffd4c0`；輔助青綠 `#0eb9b5/#e6f9f9`；灰階 `#333/#666/#999/#ccc/#ebebeb/#f5f5f5/#fff`；頁尾深灰 `#404040/#525252`。
- Header：白底、56px 高、Logo 左側，導覽含「找基金、優惠 hot、教學/FAQ」，中間有基金搜尋，右側為紀錄、錢包、登入、開戶(含 TISA)。
- 基金列表：目前以「基金照妖鏡」散佈圖與右側同類組 CP 值排名並排；基金卡含排名、基金名稱、標籤、CP 值、近 3 月走勢、多期間報酬與「快速換入 / 申購 0 手續費」CTA。
- 排序方式：報酬率、最新淨值、配息、換入資訊。
- CTA：每張基金卡底部有申購 CTA，但視覺層級被大量數字稀釋。
- Mobile：原頁在窄版資訊密度偏高，列表和圖表不容易快速判斷下一步。

## Brand Style Guide

- 保留白色 Header、淺灰頁面底、白色內容卡、深灰 Footer。
- 主 CTA 使用 FundSwap 原橘色，不新增陌生品牌色。
- 標籤使用原站青綠底與青綠字，交易/優惠狀態使用淺橘底。
- 圓角維持 4-8px，避免大圓角與模板化卡片感。
- 字級維持原系統節奏：正文 16px、輔助 13-14px、頁面 H1 約 36px、區塊標題 18-20px。
- Logo 與頁尾使用 FundSwap 現有公開資產連結。

## 可保留與需重構

可保留：
- 基金照妖鏡概念，因為它是 FundSwap 的差異化探索工具。
- CP 值排名、近 3 月走勢、多期間報酬、TISA/國人持有/主基金標籤。
- 申購 0 手續費的主 CTA 與「快速換入」交易語言。
- Header/Footer 的品牌體驗與信任聲明。

需重構：
- 首屏缺少明確投資決策入口，使用者先看到圖表和大量數字。
- CTA 分散在卡片底部，使用者不知道先比較、加入篩選還是直接申購。
- 篩選條件偏工具列，沒有對應使用者目的。
- 重複資訊過多，基金卡需要把「為什麼先看這檔」說清楚。
- Mobile 需要先顯示推薦與目的篩選，再展開完整數據。

## 專家診斷回饋

A｜Fintech UX Designer：最舊的是表格化密度與圖表優先。圖表有特色，但缺少「我該怎麼用」的可操作引導。

B｜產品負責人：申購 CTA 有顏色但沒有決策前置。使用者還沒理解差異就被要求申購，轉化會弱。

C｜金融文案：文案偏功能名詞，缺少「這檔為什麼值得先看」的短句。

D｜Funnel 分析官：流失點是首屏認知成本。200 筆結果、圖表、篩選、排名同時出現，選擇焦慮高。

E｜SEO/Growth：基金頁內容有資料價值，但缺少可被理解的段落與比較語境，停留時間依賴使用者自己解讀。

F｜前端架構師：原站 Tailwind utility 結構清楚，但列表資訊要更模組化，Mobile 應用漸進揭露。

## 轉化率問題分析

- 使用者為什麼停留：不是因為看到 200 筆基金，而是因為能快速找到「符合目的」的 3-5 檔候選。
- 現在 CTA 弱：CTA 在卡片底部，但卡片中間的數據沒有轉譯成行動理由。
- 下一步不清楚：頁面同時提供搜尋、排序、篩選、圖表、排名、申購，沒有主路徑。
- 沒人在一開始細看全部期間報酬：3 秒內應先看 CP、近 3 月、交易狀態、是否 TISA，再展開更多。
- 應刪減：首屏過多重複走勢、多欄數字、未被解釋的圖表點位、弱化的篩選按鈕。

## 入口體驗診斷

A｜消費者心理學家：使用者不敢點，因為頁面一開始要求他們理解基金資料，而不是先回應「我適合什麼」。金融決策需要安全感，安全感來自被分類、被引導、被告知下一步。

B｜Growth PM：原入口太像資料庫。搜尋、排序、篩選、圖表、排名同時出現，會讓新戶先感覺自己不懂，進而延後開戶或申購。

C｜金融文案專家：缺少情境語言。基金頁不應只說 CP 值與報酬率，也要說「想要現金流」「不想每天緊張」「第一次買基金怕選錯」。

D｜CRO 分析師：最弱的是首屏 CTA 前沒有使用者分流。CTA 要提高，必須先讓使用者選一個低壓入口，再給候選清單。

E｜Fintech UX 設計師：目前頁面工具很多，但導覽路徑不夠像一個投資助理。新版要讓卡片、問答、人格、推薦清單形成連續動線。

F｜金融法規風控顧問：不能暗示保證獲利。入口文案要用「候選、適合研究、感受、風險等級」而不是承諾報酬。

## 高轉化辯論結果

D 質疑：五個分類會不會太多？結論是可以，但必須控制在單列卡片、短文案、同一套 CTA，避免再變成另一個資料庫。最能提高 CTA 的入口不是「最大化收益」，而是「新手低風險」與「AI 智能配置」，因為它們降低選擇壓力。

B 質疑：哪個設計真的能提高開戶率？結論是「問答式入口 + 開戶 CTA」比單純基金排名更有效，因為使用者先投入 3 題回答後，心理上更願意看推薦結果。最容易讓人想點的文案是「先求看得懂，再求報酬」與「想讓系統先幫我縮小範圍」。

保留五類入口，但預設選中「穩健成長」，因為它兼顧大多數使用者的安全感與投資動機；高風險的「最大化收益」只作為明確選項，不作為預設。

## 高轉化入口 UI

- 高股息：想要現金流比較有感。適合重視配息、想定期看到回饋的人。風險 3/5。投資感受是安心感較高，但仍要接受淨值上下波動。
- 穩健成長：希望慢慢累積，不想每天緊張。適合有基本投資經驗、想長期放大資產的人。風險 3/5。投資感受是節奏穩、比較容易持續投入。
- 最大化收益：願意承受波動，換取更高機會。適合可接受明顯波動、追求資產加速成長的人。風險 5/5。投資感受是刺激感高，必須能承受回落。
- AI 智能配置：想讓系統先幫我縮小範圍。適合不知道怎麼選、想先拿到候選清單的人。風險 4/5。投資感受是選擇壓力低，但仍需自己確認風險。
- 新手低風險：先求看得懂，再求報酬。適合剛開戶、第一次買基金、怕選錯的人。風險 2/5。投資感受是壓力較低，適合建立第一筆投資經驗。

入口形式：
- 卡片式入口：五張入口卡，Hover 有原品牌橘色邊框與低陰影，點擊後更新候選基金。
- 問答式入口：三題低壓問題，回答後自動切換投資人格。
- 投資人格入口：顯示目前人格、風險 meter 與 CTA。
- Mobile UI：五張卡改成單欄，問答選項可換行，CTA 保持在卡片內。
- CTA 強化：從「看基金」改成「看高股息候選 / 看新手友善基金 / 啟用智能排序」。

## 基金頁診斷

A｜Fintech UX 大師：基金卡看起來像 Excel，因為每張卡主要在比數字，缺少「投資後會發生什麼」的理解區。使用者看到 +300% 不會立刻安心，反而會問是否已經太晚。

B｜Hedge Fund 顧問：最需要的是把收益和風險放在同一個視角。只給報酬率會讓人誤判，必須同步顯示波動、配置比例與持有情境。

C｜行為心理學家：使用者不會因為基金代碼產生行動，而是因為能想像自己的生活情境，例如每月投入、季度檢查、把它放在核心或衛星配置。

D｜數據分析官：應刪的不是所有數據，而是首層重複的長週期欄位。3 秒內保留 CP、近 3 月、1 年、走勢即可；其他資訊放進展開區。過多欄位會降低 CTA，因為使用者會覺得自己還沒看完，不能下決定。

E｜高端金融品牌設計師：展開區不能做成花俏簡報，應維持 FundSwap 的白卡、灰線、橘 CTA、青綠資訊標籤，讓專業感提升但品牌不跳掉。

F｜前端動畫工程師：互動應該是點卡片展開，不要堆很多小按鈕。Hover 只做低陰影與位移，避免金融頁看起來像遊戲。

## 基金展開區設計

- Dynamic Expand Card：點選基金卡後，在原位置展開理解區，避免跳頁打斷決策。
- 收益圖表：用 3 月、6 月、1 年、2 年長條圖讓使用者理解可能收益來源。
- 風險等級：用 1-5 格視覺化，搭配一句風險說明。
- 投資情境：用人話描述適合怎麼持有，而不是堆疊術語。
- AI 建議配置：用核心、衛星、現金比例提示它在投資組合中的位置。
- FAQ：回答一次投入、是否每天看淨值、何時停下來重看。
- 真實投資故事：讓使用者想像投資後的生活節奏。
- Sticky CTA：展開區底部固定顯示加入比較/快速換入與申購 0 手續費。

## AI 感與金融黑話清理

太像銀行的文字：
- 「投資人格入口」改成「目前比較適合你」。
- 「用投資目的篩選」改成「先用目的來篩選」。
- 「申購 0 手續費」在展開 CTA 改成「我想開始小額投資」。

太像法規的文字：
- 「申購前仍需確認公開說明書與自身風險屬性」改成「下單前，請確認公開說明書與自己的風險承受度」。
- 「過去績效不代表未來績效保證」保留意思，但改成「過去績效不代表未來一定相同」。

太難懂的詞：
- 「CP 值」在主要卡片改成「報酬/波動」。
- 「風險等級」在使用者入口改成「波動」或「你要承受的起伏」。
- 「核心 / 衛星配置」改成「主要投資 / 成長嘗試 / 保留現金」。
- 「Dynamic Expand Card」改成「這檔基金在做什麼」。

沒人想點的 CTA：
- 「查看適合我的基金」改成「看我的候選基金」。
- 「展開理解」改成「看懂這檔」。
- 「加入比較」改成「先存起來比較」。

## 文案對抗式辯論

B｜消費者心理學家質疑：
- 「這句話真的有人想點？」如果 CTA 像任務指令，例如「套用條件」「展開理解」，使用者不會有情緒動機。改成「幫我縮小選擇」「看懂這檔」，比較接近他心裡正在想的話。
- 「是否太像推銷？」如果直接說「申購」，會讓新手覺得被推著下單。展開區改成「我想開始小額投資」，保留行動但降低壓力。
- 「是否缺乏真實感？」單純說高報酬不真實。加入「如果短期下跌，你還能不能照原計畫持有」這類句子，使用者才會覺得平台站在他這邊。

## 最佳 CTA

最佳主 CTA：`幫我縮小選擇`

原因：
- 它沒有推銷感。
- 它直接回應使用者進基金頁時最大的焦慮：選項太多。
- 它比「立即申購」更適合頁面上方，因為使用者還沒準備好下單。

最佳基金卡 CTA：`看懂這檔`

原因：
- 它承認使用者還在理解階段。
- 它比「查看詳情」更有人味。
- 它能自然導向收益、風險、情境、FAQ 與 Sticky CTA。

最佳下單前 CTA：`我想開始小額投資`

原因：
- 它降低第一次投資的心理門檻。
- 它不承諾獲利，也不硬推申購。
- 它讓使用者感覺自己保有控制權。
