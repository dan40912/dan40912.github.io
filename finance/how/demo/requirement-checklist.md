# FundSwap 面試作業要求對照檢查表

審查日期：2026-05-29  
審查角色：Requirement Mapping Agent / UX Evaluation Agent / Finance Scenario Agent / Originality Agent / QA Agent

## 總評

目前 Demo 已可作為完整面試作品集展示。它不是單純視覺美化，而是圍繞同一個產品判斷展開：

> FundSwap 的問題不是基金資料不夠，而是使用者缺少把資料轉成投資決策的路徑。

完成度評分：92 / 100

扣分原因主要不是功能缺失，而是原本作品集首頁沒有明確的「逐條對照題目要求」證據頁。此缺口已補上 `/demo/07-requirement-mapping.html`。

## 任務 1：「找基金」與「基金照妖鏡」頁面優劣分析是否完成

狀態：已完成

對應交付：

- `/demo/01-audit-report.html`
- `/demo/index.html`
- `/demo/04-new-find-funds.html`
- `/demo/03-new-fund-mirror.html`

已覆蓋要求：

- UI 設計邏輯：已分析視覺層級、表格/卡片、篩選器、CTA。
- 資訊呈現效率：已指出找基金像資料庫，對新手缺少優先順序。
- 使用者操作路徑：已指出使用者從篩選、比較到猶豫之間缺少承接。
- 視覺化表達：已分析基金照妖鏡散點圖方向正確，但 CP 值需要被翻譯。
- 金融決策輔助性：已加入「為什麼適合你」「主要風險」「適合情境」等決策語言。

審查判斷：

- 合格，而且有具體觀點。
- 不是只有說「優化 UX」，而是明確指出「資料完整但決策路徑不足」。

## 任務 2：「基金照妖鏡」0 到 1 複盤與優化是否完成

狀態：已完成，並已補強成明確對照頁

對應交付：

- `/demo/03-new-fund-mirror.html`
- `/demo/07-requirement-mapping.html`

已覆蓋要求：

- 明確回答：會保留散點圖，但不讓散點圖當第一步。
- 原因：散點圖適合比較，不適合新手第一眼決策。
- 新流程：
  - Step 1：先選投資需求 / 投資人格。
  - Step 2：系統切換判斷依據。
  - Step 3：再用報酬率 × 波動度四象限解讀基金。
  - Step 4：基金詳情卡與比較表承接下一步。
- 保留 CP 值概念，但用「報酬 / 波動」說明，避免暗示投資推薦。

審查判斷：

- 合格。
- 07 頁補強了「如果從 0 到 1 主導，我會怎麼設計」的文字證據。

## 任務 3：1-2 個創新功能是否完成

狀態：已超額完成

對應交付：

- `/demo/05-fund-fit-wizard.html`
- `/demo/06-fund-arena.html`
- `/demo/04-new-find-funds.html`

創新功能 1：基金適配問答精靈

- 目標使用者：新手與半熟手。
- 核心價值：用 5 步驟問答把模糊投資需求轉成基金篩選條件。
- 差異化：不是直接推薦基金，而是顯示使用者風格、適合觀察的基金類型、適配原因與風險。

創新功能 2：Fund Arena 基金比較場

- 目標使用者：資深投資者與需要比較 2-3 檔基金的人。
- 核心價值：用雷達圖、分數條、差異標籤與情境卡比較基金。
- 差異化：不說哪檔一定更好，而是說誰比較適合穩健型、成長型、配息型情境。

加分功能：新版找基金列表頁

- 搜尋、chips、進階篩選、排序、卡片/列表切換、最多 3 檔比較。
- 每檔基金補上「為什麼適合你」與「主要風險」。

審查判斷：

- 已超過題目要求的 1-2 個創新方向。
- 創新不是脫離金融場景，而是從使用者決策需求出發。

## 專業性是否足夠

狀態：足夠

證據：

- 有 UI/UX 分析維度：視覺層級、資訊效率、操作路徑、CTA、視覺化表達。
- 有基金語意：報酬率、波動度、CP 值、風險等級、配息、費用、投資區域。
- 有投資者需求：新手焦慮、穩健、成長、現金流、退休、短中期配置。
- 有合規意識：多處標註「篩選輔助」「非投資建議」「非基金推薦」。

審查判斷：

- 比一般 UX 報告更貼近金融產品。
- 沒有只停留在排版美化。
