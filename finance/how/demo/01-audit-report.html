UX AUDIT REPORT

「找基金」與「基金照妖鏡」頁面優劣分析

本報告基於上一輪 site research，聚焦現有基金探索流程的三個問題：使用者不知道該看什麼、報酬與風險沒有被翻譯成決策語言、CTA 沒有接住猶豫中的使用者。

核心判斷 資料完整，但決策路徑不足

現站具備基金搜尋、排序、視覺化比較與交易入口，但對新手與半熟手而言，還缺少「如何從資料走到判斷」的引導層。

PAGE 01

找基金頁：像資料庫，但還不像決策工具

找基金頁對進階投資者有效，因為它提供完整篩選與排序；但對新手而言，一進來看到大量基金與欄位，會先產生「我是不是不懂」的壓力。

優點

  • 基金資料完整，包含報酬、配息、交易狀態與標籤。
  • 排序與篩選對已知目標的使用者有效。
  • Header / Footer 與 FundSwap 品牌一致，金融平台感明確。

缺點

  • 首屏像資料庫，沒有先問使用者想解決什麼問題。
  • 報酬、配息、標籤、交易 CTA 層級接近，難以快速掃描。
  • 高報酬排序容易讓新手忽略波動與最大回撤。

改善機會

  • 在列表前加「保守 / 積極 / 長期」決策入口。
  • 把基金卡第一層改成「為什麼值得看」而不是只堆數字。
  • CTA 分成加入比較、看懂詳情、交易前確認三層。

PAGE 02

基金照妖鏡：視覺化方向正確，但 CP 值需要被翻譯

官方教學將基金照妖鏡定義為用近一年報酬率與波動度找 CP 值高的基金。這個方向有差異化，但使用者仍需要知道「我該看哪個點，為什麼」。

報酬率 波動度 報酬較高 / 波動較低

目前的斷點

散點圖能讓使用者看到相對位置，但如果沒有投資人格視角，保守型、積極型、長期型會被迫看同一套排序。

保守型看 MDD 積極型看動能 長期型看穩定度

優點

用圖表把報酬與波動放在一起，比純表格更接近基金決策。

缺點

CP 值若只顯示數字，使用者不會自然理解它是「報酬 / 波動」的比較概念。

改善機會

保留散點圖，但在圖表前先建立使用者視角，再用人格解釋哪些點值得研究。

AGENT NOTES

五個 Agent 的結論

UI Design Agent

不要重做品牌。應保留 FundSwap 白底、灰線、橘 CTA，調整卡片內的優先層級。

UX Flow Agent

目前流程從資料開始，建議改成從投資視角開始，降低進站後的選擇焦慮。

Financial UX Agent

CP 值、波動度、MDD 都合理，但要翻譯成人話，且不能暗示個別投資建議。

Content Strategy Agent

把「CP 值高」改成「同樣波動下，過去報酬相對較好」，讓使用者知道指標含義。

Frontend Agent

目前靜態架構足夠展示，下一步可直接在 `output/index.html` 線路延伸，不需導入框架。

NEXT DESIGN MOVE

下一版 Demo 的優先改動

  1. 01 先選投資視角

    在基金列表前加入保守、積極、長期三種入口，讓使用者知道該用哪組指標看基金。

  2. 02 重寫基金卡第一層

    第一眼不要只給報酬率，應回答「為什麼這檔值得研究」。

  3. 03 重構 CTA 階段

    下單前先提供加入比較、看懂詳情、查看風險，避免過早交易感。
