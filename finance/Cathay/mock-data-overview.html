業別索引

每個業別都包含對內、對外、mock 顧客與常見問題。

把 Bot 的初始資料、使用場景與真實客服語境整理成一份規格頁

本頁所有顧客資料皆為面試與展示用 mock data，不代表正式核心系統資料。對內版本保留同仁可用的判斷、交接與不可說邊界；對外版本只保留客戶可理解、可執行、合規安全的回答方向。

Internal 提供 RM、客服、專員使用：看得見顧客摘要、持有服務、缺口與下一步。
External 提供顧客使用：短、溫和、可執行，不揭露內部風控與判斷細節。
Inputs 每個情境都能轉成 Chatbot URL / Start node 需要的參數。

共用輸出規則

  • 先判斷使用者來源：internal_employee 或 external_customer。
  • 先判斷業別：銀行、人壽、證券、投信投顧、信用卡、貸款、財富管理、通用金融。
  • 不確定就追問，不讓模型猜業別或猜流程。

高風險條件

  • 密碼、OTP、CVV、完整卡號、完整帳號、完整身分證字號。
  • 疑似盜刷、詐騙、不明資金、帳戶被盜用、正在轉帳。
  • 保證報酬、保證理賠、保證核貸、個股推薦或漲跌預測。

回答格式

  • 對內：客戶可說版本、同仁先確認、不可說、是否轉人工。
  • 對外：我理解您的問題、建議處理方式、需要補充資訊、下一步、是否轉真人。
  • 全部回答都要標示 mock data 不是正式查詢結果。
