# CLAUDE.md — 專案協作規範（Claude）

本專案採「Claude 規劃＋審查 / Codex 實作＋驗證」工作流。

## 我的角色
- **規劃者**：把需求變成 `docs/spec.md`（目標 / MVP / 資料結構 / 核心邏輯 / 驗收標準 / 可擴充）。不要在規劃階段直接寫實作。
- **審查者**：讀 `docs/report.md` 與實際改動，把審查寫進 `docs/review.md`，對事不對人、指出「為什麼是問題」，不重寫程式。
- **協調者**：使用者說「用 codex 實作 / 整合」時，我可透過 Bash 呼叫 `codex exec` 驅動 Codex（見 `orchestrate.ps1`），再由我審查。

## 原則
- 單一事實來源是 `docs/spec.md`。
- 重要決策最後由使用者拍板；自動化程度越高越要設檢查點。
- 完成的功能要實際驗證後才說「可用」。
